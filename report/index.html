実装内容

1. 標準 FFmpeg 前処理を固定 office preset へ変更

python/whisper_server.py の runtime filter chain を以下に固定しました。

highpass=f=120,lowpass=f=6800,afftdn=nf=-28:tn=1

この構成は、背景話者や非人声音ノイズを持ち上げる dynaudnorm と auto gain を避け、軽い帯域制限と stationary denoise のみを標準処理として適用します。afftdn が使えない環境では highpass=f=120,lowpass=f=6800 にフォールバックします。

2. 低活動 gate と低信頼 gate の二段構成

既存の audio activity analysis による低活動スキップを維持し、その後段に Whisper segment metrics の gate を追加しました。

  • no_speech_prob >= 0.6 を抑制
  • avg_logprob <= -0.8 を抑制
  • compression_ratio >= 2.4 を抑制

CPU faster-whisper と MLX Whisper の両方から segment metrics を収集できるようにし、既存の BackendManager.transcribe(...) 互換 API は残しつつ、新しく transcribe_with_details(...) で metrics 付き結果を扱う構成にしています。

3. Apple Voice Processing をライブ録音入口で標準有効化

KotoType/Sources/KotoType/Audio/RealtimeRecorder.swift の input tap 設置前に setVoiceProcessingEnabled(true) を試行するようにしました。ユーザー設定は増やさず、macOS や入力デバイスの都合で有効化に失敗した場合は録音を止めずに従来の raw input tap へフォールバックします。

互換性調査のため KOTOTYPE_DISABLE_APPLE_VOICE_PROCESSING=1 のみを残し、batch 音声作成ログには appleVoiceProcessing=true/false を出します。

4. 評価 CLI は比較用に維持

tools/evaluate_noise_strategies.py は、今回の判断根拠を再実行できるように残しています。ただし legacy current strategy は評価 CLI 内で self-contained に再現し、runtime の実装には戻さない構造です。

削除した機能・経路

削除対象 理由 現在の扱い
dynaudnorm runtime path 遠い背景話者や非人声音ノイズの activity を持ち上げ、gate の効きを弱めるため。 評価 CLI の legacy current 比較にのみ残置。
compressor option 標準採用候補から外れ、runtime で設定分岐を持つ必要がなくなったため。 削除。
KOTOTYPE_ENABLE_NOISE_REDUCTION ユーザー向け/運用向けモード切替を増やさない issue 方針に反するため。 削除。互換デバッグは KOTOTYPE_SKIP_AUDIO_PREPROCESSING のみ。
KOTOTYPE_AUTO_GAIN_ENABLED と KOTOTYPE_AUTO_GAIN_* auto gain は背景話者を増幅し得るため、標準処理として不採用。 削除。
determine_gain_for_weak_audio, apply_gain_to_wav, analyze_wav_peak_dbfs auto gain runtime path の削除により不要。 削除。評価 CLI は独自 helper で legacy 比較を再現。

残るリスクと次の確認

  • 近距離の背景話者のみ音声は、Python 側 gate だけでは完全には抑制できません。Apple Voice Processing は実装済みですが、実オフィス環境、Mac 内蔵マイク、口元マイクでの dogfooding が必要です。
  • 評価データは再現可能な合成データ中心です。Apple Voice Processing は入力デバイス依存のため、同一環境での raw 相当比較と実録評価が残ります。
  • Swift IntegrationTests は実機マイクや実 Python worker 起動へ依存しない fixture に整理済みです。実オフィス環境での最終 dogfooding は別途必要です。
